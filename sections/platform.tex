The BHTs generated by BibleGoAI are served to end users through \textbf{BibleGo.org}, a web application built with Next.js \citep{nextjs}, React, and TypeScript, backed by a PostgreSQL database and hosted on Vercel. The platform currently serves hundreds of users per month and provides three primary modes of interaction.

\paragraph{Commentary reading.}
The main reading interface presents a responsive three-column layout (Figure~\ref{fig:desktop-bht}): a book selector on the left, the Bible text (NASB) in the center with inline original-language tags, and a commentary panel on the right. When a user selects a verse, the corresponding BHT is displayed along with footnotes that link each sentence to its source commentary. Users can expand individual footnotes to view the original quotes, the commentator attribution, and a similarity score indicating the strength of the connection. Each commentator's full excerpt is also available as a separate card, enabling side-by-side comparison of perspectives. On mobile devices, the commentary panel appears as a slide-up drawer (Figure~\ref{fig:bht-mobile}).

\paragraph{Word study.}
BibleGo.org includes an interactive word study feature---the \textit{Word Donut}---powered by Strong's concordance data built from the original language tagging pipeline (Section~\ref{sec:architecture}). Users can select any word in the Bible text to view its original Greek or Hebrew form, transliteration, and definition. An interactive doughnut chart then visualizes how the NASB translators chose to render that word across all its occurrences in the Bible, showing each English translation variant and its frequency (Figure~\ref{fig:word-donut}). This feature supports pagination and filtering by translation variant, enabling deep original-language exploration without requiring knowledge of Greek or Hebrew. As the application notes, drawing from Edward W. Goodrick: while the various English renderings can enrich understanding, ``we are enslaved to the tyranny of the context''---each usage must be understood within its own passage.

\paragraph{Full commentary browsing.}
Users can switch from the BHT summary view to a full commentary mode, which displays the complete chapter-level commentary for any individual commentator (Figure~\ref{fig:full-commentary}). This enables readers who encounter an interesting insight in a BHT to dive deeper into the full context of that commentator's work.

\paragraph{User experience.}
The application features dark mode support, adjustable font sizes (persisted across sessions), mobile-responsive design with a drawer-based layout for smaller screens, deep linking via URL parameters (e.g., \texttt{/read?v=Matthew.1.1}), and an in-app feedback system with structured categories. The UI underwent four rounds of dedicated usability testing---including tests focused specifically on the drawer navigation and the word study tool---with feedback directly informing design iterations.

\paragraph{Data flow.}
Pre-generated BHTs from the BibleGoAI pipeline are loaded into a PostgreSQL database. The Next.js application queries this database through server-side API routes, returning BHT text, footnotes, scoring metadata, and commentator information. Bible text with Strong's number tags is served from static JSON files, while Strong's concordance entries are queried from the database. This hybrid approach balances query flexibility with serving performance.
